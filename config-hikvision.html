1. Qué producto de Hikvision usa Limen

⚠️ Importante: Hikvision tiene dos APIs completamente distintas. Limen usa ISAPI (device-level), no HikCentral OpenAPI. Elegir la incorrecta es el error más frecuente en la configuración.
CaracterísticaISAPI (lo que usa Limen)HikCentral OpenAPI (no usado)
¿Qué es? API REST embebida en cada dispositivo físico (controladora DS-K2xxx, panel ACS, NVR, etc.) API del software de gestión centralizada HikCentral Professional
Autenticación HTTP Digest (RFC 7616) — usuario + contraseña del dispositivo AppKey + AppSecret con firma HMAC-SHA256 por request
URL base IP o hostname directo del dispositivo: http://192.168.1.64 Servidor HikCentral: https://hkc.empresa.com
Requiere software extra No. La API está en el firmware del dispositivo. Sí. HikCentral Professional instalado y con licencia activa.
Credenciales en Limen Usuario y contraseña del dispositivo (ej: admin / Admin1234) AppKey y AppSecret generados en HikCentral → Open Platform
Dispositivos compatibles DS-K2800 series, DS-K1T671 series, DS-K3 series, NVR con módulo ACS, DS-K2604T, etc. Cualquier dispositivo gestionado por HikCentral

2. Prerequisitos

  1. Dispositivo Hikvision con módulo ACS habilitado. La serie DS-K2xxx (controladoras de acceso) tiene ISAPI nativo. En NVRs/DVRs, verificar que el firmware tenga el módulo Access Control activo.
  2. Firmware compatible. ISAPI v2.0+ (firmware ≥ V1.5 en la mayoría de DS-K series). En la interfaz web del dispositivo: Mantenimiento → Información del Dispositivo → versión de firmware.
  3. Acceso de red desde el servidor Limen al dispositivo. El servidor donde corre Limen debe poder alcanzar la IP del dispositivo en el puerto 80/443 (HTTP/HTTPS). Si el dispositivo está en otra VLAN, verificar ACLs de red.
  4. Usuario con permisos de Remote Control y Access Control. No es necesario usar admin; se puede crear un usuario dedicado limen con solo los permisos necesarios: Remote Control (apertura remota) y Access Control (gestión de usuarios).

3. Habilitar ISAPI en el dispositivo

La mayoría de dispositivos Hikvision tienen ISAPI habilitado por defecto. Para verificar y habilitar:

Vía interfaz web del dispositivo

  1. Acceder a http://<IP-del-dispositivo> en el navegador
  2. Ir a Configuración → Red → Configuración avanzada → Integración
  3. Activar ISAPI si no está marcado
  4. Guardar y reiniciar si se modificó

Vía iVMS-4200 (software de gestión local)

  1. Seleccionar el dispositivo → Configuración remota
  2. Red → Configuración avanzada → Activar ISAPI
💡 Tip: Verificar que ISAPI responde antes de configurar en Limen: 
curl -u admin:TuContraseña --digest http://192.168.1.64/ISAPI/System/deviceInfo
Si devuelve un JSON o XML con información del dispositivo, ISAPI está activo y las credenciales son correctas.

4. Configurar en Limen

  1. Ir a Administración → Integraciones en Limen
  2. En la tarjeta Control de Acceso Físico, seleccionar proveedor Hikvision ISAPI
  3. Completar los campos:
    CampoDescripciónEjemplo
    URL Base IP o hostname del dispositivo Hikvision. Si el dispositivo usa HTTPS con certificado auto-firmado, usa https:// (Limen acepta certificados auto-firmados en modo dispositivo). http://192.168.1.64
    https://hik.empresa.local
    Usuario Usuario del dispositivo (admin u otro con permisos de AC). No es un AppKey de HikCentral. admin o limen
    Contraseña Contraseña del usuario del dispositivo. Se cifra con DataProtection antes de persistir. Admin1234!
  4. Hacer clic en Probar conexión para verificar que las credenciales son válidas
  5. Hacer clic en Guardar

5. Endpoints ISAPI Implementados

Esta es la tabla exacta de qué endpoint ISAPI usa cada método de Limen:

Método LimenMétodo HTTPEndpoint ISAPINotas
SyncPersonAsync POST /ISAPI/AccessControl/UserInfo/Record?format=json Crea o actualiza la persona. Si el employeeNo ya existe, lo actualiza. Si se provee foto, se hace una segunda llamada a /ISAPI/Intelligent/FDLib/FaceDataRecord (no crítico; si el dispositivo no tiene reconocimiento facial, no falla).
BlockUserAsync PUT /ISAPI/AccessControl/UserInfo/SetUp?format=json Establece Valid.enable = false. Intento alternativo con /ISAPI/AccessControl/UserInfo/Modify si SetUp falla (compatibilidad con firmware antiguo).
UnblockUserAsync PUT /ISAPI/AccessControl/UserInfo/SetUp?format=json Establece Valid.enable = true. Mismo fallback.
AssignAccessProfileAsync PUT /ISAPI/AccessControl/UserInfo/SetUp?format=json El profileName se interpreta como planTemplateNo numérico. Si no es numérico, se usa "1" (plan por defecto). Los planes de acceso deben estar pre-configurados en el dispositivo.
RemoteOpenDoorAsync PUT /ISAPI/AccessControl/RemoteControl/door/{doorNo} Body XML: <RemoteControlDoor><cmd>open</cmd></RemoteControlDoor>. El doorHardwareId debe ser el número de puerta (entero, empieza en 1). Este es el único endpoint que usa XML en lugar de JSON.
RegisterVehiclePlateAsync PUT /ISAPI/AccessControl/UserInfo/SetUp?format=json Usa el campo vehicleList dentro de UserInfo. Solo soportado en dispositivos DS-K2 series y algunos DS-K1 series con firmware reciente. En dispositivos sin soporte devuelve error no crítico (se registra warning en logs, no falla la operación). Para LPR completo se recomienda configurar la lista blanca directamente en el dispositivo o usar HikCentral.
GetAccessEventsAsync POST /ISAPI/AccessControl/AcsEvent?format=json Paginado interno (50 eventos por página). Máximo 1000 eventos por llamada. Filtra por rango de tiempo con startTime/endTime en ISO 8601. majorEventType=5 = evento de control de acceso; subEventType=75 = acceso concedido, 76 = denegado, 35 = salida local.
GetHardwareDoorsAsync GET /ISAPI/AccessControl/Door/param?format=json (intento 1)
/ISAPI/AccessControl/capabilities?format=json + iteración (fallback) 		Primero intenta obtener la lista completa. Si ese endpoint no existe en el firmware, consulta las capacidades del dispositivo para saber cuántas puertas tiene y las itera individualmente con GET /ISAPI/AccessControl/Door/param/{doorNo}. Si tampoco hay capabilities, itera sobre 4 puertas como default conservador.

6. Mapeo de Puertas (HardwareId)

En Limen, cada puerta tiene un campo Hardware ID que es el identificador en el sistema de control de acceso físico. Para Hikvision ISAPI, ese ID debe ser el número de puerta entero (1, 2, 3...) como está configurado en el dispositivo.

¿Cómo saber el número de cada puerta?

  1. En Limen: ir a Administración → Puertas → Importar desde hardware. Limen consulta el dispositivo con GetHardwareDoorsAsync y lista las puertas disponibles con su número y nombre para que las mapees a las zonas de acceso.
  2. Alternativamente, en la interfaz web del dispositivo: Acceso → Puerta muestra la lista con los números de puerta.
  3. Vía curl para verificar directamente:
    curl -u admin:Contraseña --digest \
      "http://192.168.1.64/ISAPI/AccessControl/Door/param/1?format=json"
⚠️ Importante: Para RemoteOpenDoorAsync el doorHardwareId debe ser un entero ≥ 1. El string "1", "2", etc. Limen valida esto antes de llamar al endpoint — si envías un UUID o un string no numérico, la operación falla con un mensaje claro antes de hacer ninguna request al dispositivo.

7. Cómo funciona la autenticación HTTP Digest

ISAPI usa HTTP Digest Authentication estándar (RFC 7616). A diferencia de Basic Auth, las credenciales nunca viajan en texto plano: el servidor responde al primer request con un reto (nonce) y el cliente responde con un hash MD5/SHA-256 de las credenciales + el nonce.

En .NET, HttpClientHandler maneja esto automáticamente cuando se le configuran Credentials = new NetworkCredential(user, pass). Limen establece además PreAuthenticate = true para evitar el round-trip inicial en llamadas subsecuentes (mejora de latencia en operaciones consecutivas).

Certificados auto-firmados

La mayoría de dispositivos Hikvision usan certificados auto-firmados cuando se configura HTTPS. Limen acepta cualquier certificado de dispositivo en modo ISAPI (DangerousAcceptAnyServerCertificateValidator). En producción, lo correcto es instalar un certificado firmado por un CA corporativo en el dispositivo y configurar el trust store del servidor Limen, o usar HTTP en una VLAN aislada.

8. Verificación manual con curl

Antes de configurar en Limen, verificar manualmente que las credenciales funcionan:

Verificar conectividad y credenciales

# Información del dispositivo
curl -u admin:Contraseña --digest \
  "http://192.168.1.64/ISAPI/System/deviceInfo?format=json"

Crear un usuario de prueba

curl -u admin:Contraseña --digest -X POST \
  -H "Content-Type: application/json" \
  -d '{"UserInfo":{"employeeNo":"9999","name":"Test Limen","userType":"normal","Valid":{"enable":true,"beginTime":"2000-01-01T00:00:00","endTime":"2030-12-31T23:59:59","timeType":"local"},"doorRight":"1","RightPlan":[{"doorNo":1,"planTemplateNo":"1"}]}}' \
  "http://192.168.1.64/ISAPI/AccessControl/UserInfo/Record?format=json"

Respuesta esperada: {"statusCode":1,"statusString":"OK",...}

Abrir puerta 1 remotamente

curl -u admin:Contraseña --digest -X PUT \
  -H "Content-Type: application/xml" \
  -d '<RemoteControlDoor><cmd>open</cmd></RemoteControlDoor>' \
  "http://192.168.1.64/ISAPI/AccessControl/RemoteControl/door/1"

Leer eventos de acceso del último día

curl -u admin:Contraseña --digest -X POST \
  -H "Content-Type: application/json" \
  -d '{"AcsEventCond":{"searchID":"1","searchResultPosition":0,"maxResults":20,"major":0,"minor":0,"startTime":"2025-05-16T00:00:00+00:00","endTime":"2025-05-17T23:59:59+00:00"}}' \
  "http://192.168.1.64/ISAPI/AccessControl/AcsEvent?format=json"

9. Troubleshooting

SíntomaCausa probableSolución
HTTP 401 Unauthorized Credenciales incorrectas o usuario sin permisos ISAPI Verificar usuario/contraseña con curl. Confirmar que el usuario tiene rol con permisos de Remote Control. Algunos dispositivos requieren habilitar "Digest Auth" explícitamente en configuración de seguridad.
Connection refused / timeout Puerto bloqueado por firewall o ISAPI deshabilitado Verificar conectividad: telnet 192.168.1.64 80. Verificar que ISAPI está habilitado en Configuración → Red → Integración.
statusCode: 5 (error de formato) El payload JSON enviado tiene un campo no reconocido por este firmware Revisar logs de Limen para ver el payload exacto. Algunos campos como vehicleList solo existen en ciertos modelos/firmwares. Actualizar firmware del dispositivo o usar una versión del payload más básica.
statusCode: 7 (recurso no encontrado) El employeeNo no existe al intentar SetUp Asegurarse de llamar SyncPersonAsync antes de BlockUserAsync/AssignAccessProfileAsync. La persona debe existir en el dispositivo antes de poder modificarla.
RemoteOpen retorna OK pero la puerta no abre El número de puerta es incorrecto, o el relé no está conectado Verificar que el doorHardwareId en Limen coincide con el número de puerta físico del dispositivo. Probar directamente con curl. Verificar estado del relé en la interfaz web del dispositivo.
GetHardwareDoors retorna lista vacía El endpoint de lista no existe en este firmware Limen intenta automáticamente el modo de iteración. Si la lista sigue vacía, el firmware puede ser muy antiguo. Agregar las puertas manualmente en Limen con el número correcto (entero desde 1).
Foto facial no se sube El dispositivo no tiene módulo de reconocimiento facial Comportamiento esperado. Solo dispositivos MinMoe y algunos DS-K1T671 soportan reconocimiento facial vía ISAPI FDLib. El acceso por tarjeta/PIN sigue funcionando normalmente.

10. Limitaciones conocidas

FuncionalidadEstadoNotas
Alta y baja de personas ✅ Implementado SyncPerson, Block, Unblock
Apertura remota de puerta ✅ Implementado Requiere número de puerta entero como HardwareId
Lectura de eventos de acceso ✅ Implementado Paginado hasta 1000 eventos por llamada
Listado de puertas del dispositivo ✅ Implementado Con fallback por iteración si el endpoint de lista no existe
Sincronización de foto facial ⚠️ Parcial Solo en dispositivos con FDLib (MinMoe, DS-K1T671M). En otros, se omite sin error.
Registro de patente vehicular ⚠️ Parcial Solo en DS-K2 series con campo vehicleList. Para LPR completo, configurar en dispositivo o usar HikCentral.
Asignación de perfiles de acceso ⚠️ Básico Solo por planTemplateNo numérico. Los planes de acceso deben estar pre-creados en el dispositivo; Limen no los crea.
Webhooks / eventos en tiempo real ❌ No implementado Limen usa polling periódico de AcsEvent. Hikvision ISAPI soporta eventos push vía /ISAPI/Event/notification/alertStream (multipart streaming) o HTTP notifications. Implementar si el cliente requiere latencia <30s.
Múltiples dispositivos por tenant ❌ No implementado La configuración actual es 1 dispositivo por tenant. Para multi-dispositivo (múltiples controladoras) se necesita un servidor intermedio (NVR con módulo ACS o HikCentral) que agregue todas las controladoras en un solo endpoint.
