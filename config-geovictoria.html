GEOVICTORIA INTEGRATION

Sincronización de Identidad mediante Polling

← Volver al Índice Técnico

1. Arquitectura de Integración

La integración con GeoVictoria se basa en un modelo de Polling (Consulta Periódica). Limen consulta activamente el estado de los empleados cada cierto tiempo, ya que GeoVictoria no expone Webhooks para eventos de baja en tiempo real en este conector.

2. Proceso de Autenticación

Limen utiliza un flujo de token dinámico para garantizar la seguridad de la comunicación:

  • Endpoint: POST /api/v1/Login
  • Credenciales: Requiere ConsumerKey y ConsumerSecret provistos por el equipo de integraciones de GeoVictoria.
  • Gestión de Token: El Bearer Token obtenido se cachea en Limen durante 50 minutos.
  • Failsafe: El sistema implementa Auto-retry automático en caso de recibir un error 401 (Unauthorized).

3. Sincronización de Empleados

Para obtener la nómina vigente, Limen ejecuta:

POST /api/user/list
Payload: { "status": 1 } // Filtra solo empleados activos

4. Configuración por Tenant

En el panel de Administración de Limen, debe configurar los siguientes campos obligatorios:

CampoDescripción
BaseUrlURL base de la API de GeoVictoria para su instancia.
ConsumerKeyLlave de acceso de la aplicación.
ConsumerSecretSecreto de la aplicación (se guarda cifrado).
Nota Importante: Al no contar con Webhooks, el bloqueo de bajas laborales depende de la frecuencia del job de sincronización. Para bloqueos manuales inmediatos, utilice el botón de "Bloqueo" en el perfil del usuario en Limen.
