GEOVICTORIA

Sincronización de Dotación mediante Polling Periódico

← Volver al Índice Técnico

1. Arquitectura de Integración

La integración con GeoVictoria opera mediante Polling Periódico: Limen consulta activamente la API de GeoVictoria a intervalos regulares para mantener la nómina sincronizada.

MecanismoCuándo actúaLatencia
Polling periódico Sincronización completa de nómina activa (job cada 15 min) ≤ 15 min
⚠️ Diferencia clave con Buk: GeoVictoria no soporta webhooks en este conector. Limen no puede recibir notificaciones instantáneas de bajas laborales. El bloqueo de acceso ocurre en el próximo ciclo de polling (máximo 15 minutos). Para bajas urgentes use el bloqueo manual inmediato desde Personas → [empleado] → Bloquear acceso.

2. Modelo de Autenticación

GeoVictoria usa un flujo de token dinámico: Limen se autentica con ConsumerKey + ConsumerSecret y obtiene un Bearer Token con duración de 60 minutos.

Flujo de autenticación

POST https://[instancia].geovictoria.com/api/v1/Login
Content-Type: application/json

{
  "ConsumerKey": "{tu_consumer_key}",
  "ConsumerSecret": "{tu_consumer_secret}"
}

// Respuesta:
{
  "token": "eyJhbGciOiJIUzI1NiJ9...",
  "expiration": "2026-05-17T15:30:00Z"
}
ComportamientoDetalle
Cache del token Limen cachea el token por 50 minutos (conservador respecto al TTL de 60 min de GeoVictoria) para evitar logins innecesarios en cada ciclo
Auto-retry en 401 Si GeoVictoria responde 401 (token expirado antes de tiempo), Limen limpia el cache, obtiene un token nuevo y reintenta la consulta automáticamente — sin intervención manual
Alcance por tenant La clave de cache incluye el ConsumerKey para evitar colisiones entre tenants que compartan el mismo servidor Limen

3. Obtener Credenciales en GeoVictoria

El ConsumerKey y ConsumerSecret son credenciales de integración de aplicación, distintas al usuario/contraseña de acceso al portal.

  1. Ingresar al panel de GeoVictoria como administrador
  2. Ir a Configuración → Integraciones → API / Aplicaciones externas
  3. Crear una nueva aplicación con nombre descriptivo: "Limen - Control de Acceso"
  4. El sistema generará el ConsumerKey y el ConsumerSecret
  5. Copiar ambos valores — el ConsumerSecret generalmente no se vuelve a mostrar
Si no aparece la sección de API: Esta funcionalidad puede requerir habilitación por parte del equipo de GeoVictoria. Contactar al soporte de GeoVictoria indicando que necesitas acceso a la API REST de usuarios.

4. Configurar la Integración en Limen

  1. Ingresar a Limen como TenantAdmin
  2. Ir a Administración → Integraciones → RRHH → GeoVictoria
  3. Completar los campos:
CampoDescripciónEjemplo
URL Base URL raíz de la instancia GeoVictoria del cliente, sin barra final https://miempresa.geovictoria.com
Consumer Key Llave de la aplicación generada en el paso 3 gv_ck_aBcDe...
Consumer Secret Secret de la aplicación. Se cifra con DataProtection antes de guardarse. gv_cs_XyZw...
  1. Hacer clic en Guardar
  2. Hacer clic en Probar Conexión. Debe mostrar: "Conexión exitosa. N empleados activos."
Seguridad: El ConsumerSecret se cifra con AES-256 (DataProtection) antes de almacenarse en la base de datos. No se guarda en texto plano ni en appsettings. Una vez guardado, el campo queda en blanco en el formulario — eso es normal.

5. Endpoints de la API GeoVictoria (Referencia Técnica)

Limen usa los siguientes endpoints de GeoVictoria (verificados mayo 2026):

→ Autenticación interna 		Obtiene el Bearer Token de sesión. Se llama automáticamente cuando el token no está en cache o expiró. {"ConsumerKey": "...", "ConsumerSecret": "..."}
POST /api/user/list
→ GetActiveEmployeesAsync()
→ GetTerminatedEmployeesAsync()
→ GetEmployeeByRutAsync(rut)
→ TestConnectionAsync() 		Retorna todos los usuarios de la cuenta GeoVictoria. Limen filtra localmente por status:
  • status == 1 → activos
  • status != 1 → desvinculados

Nota: GeoVictoria no expone endpoints separados para empleados activos o inactivos. La API devuelve la lista completa y Limen aplica el filtro en memoria. Para búsqueda por RUT (GetEmployeeByRutAsync), Limen descarga toda la nómina activa y filtra localmente — esto es una limitación de la API de GeoVictoria.

6. Flujo de Sincronización Periódica

El job de sincronización (IdentitySyncJob) corre cada 15 minutos y ejecuta:

EtapaAcciónTiempo est.
1. Token Obtener Bearer Token desde cache o autenticar contra GeoVictoria 0–500ms
2. Nómina POST /api/user/list → lista completa de usuarios 1–5s (depende del tamaño)
3. Diff Comparar nómina GeoVictoria con EmployeeRecord en Limen: detectar altas, bajas y cambios ~200ms
4. Bajas Para empleados que pasaron de activo a inactivo → bloqueo en hardware (ZK/Hikvision) vía IAccessControlPlatformClient 2–5s por empleado
5. Auditoría Registro de cada cambio en AuditLog ~50ms
Baja urgente — no esperes el próximo ciclo: Si un empleado es desvinculado y necesitas bloquear su acceso de inmediato, usa el bloqueo manual: Personas → [nombre del empleado] → Bloquear acceso. Esto actúa directamente sobre el hardware sin esperar el ciclo de 15 minutos.

7. Verificar la Integración

7.1 Verificar conexión desde la UI

  1. Ir a Administración → Integraciones → RRHH → GeoVictoria
  2. Hacer clic en Probar Conexión
  3. El resultado debe mostrar el número de empleados activos. Si aparece un error, revisar la sección Troubleshooting

7.2 Verificar autenticación manualmente con curl

# Paso 1: Obtener token
curl -X POST https://[instancia].geovictoria.com/api/v1/Login \
  -H "Content-Type: application/json" \
  -d '{"ConsumerKey":"TU_KEY","ConsumerSecret":"TU_SECRET"}'

# Paso 2: Consultar usuarios con el token obtenido
curl -X POST https://[instancia].geovictoria.com/api/user/list \
  -H "Authorization: Bearer {TOKEN_DEL_PASO_1}" \
  -H "Content-Type: application/json" \
  -d '{}'

La respuesta del paso 2 debe ser un array JSON de usuarios. Si recibes 401 en el paso 1, las credenciales son incorrectas. Si recibes 401 en el paso 2, el token del paso 1 no fue copiado correctamente.

7.3 Verificar en AuditLog de Limen

  1. Ir a Administración → Auditoría
  2. Buscar eventos con tipo GV_SYNC_COMPLETED para ver el historial de sincronizaciones
  3. Si hay errores, buscar GV_SYNC_ERROR con el detalle del problema

8. Troubleshooting

SíntomaCausa probableSolución
"Probar Conexión" devuelve error 401 ConsumerKey o ConsumerSecret incorrectos Regenerar las credenciales en GeoVictoria y actualizar en Limen
"Probar Conexión" devuelve error de conexión / timeout La URL Base apunta a un servidor incorrecto o hay un firewall bloqueando Verificar que la URL no tenga barra final ni rutas adicionales. Probar con curl desde el servidor de Limen
La nómina se sincroniza pero los empleados no aparecen en Limen El campo identifier (RUT) viene vacío o con formato incorrecto desde GeoVictoria Revisar en GeoVictoria que los empleados tienen RUT cargado. El campo identifier es el identificador principal
Un empleado desvinculado sigue con acceso después de 15 minutos El campo status del empleado no cambió a 0 en GeoVictoria, o el job falló silenciosamente Verificar el status del empleado en GeoVictoria. Revisar logs de IdentitySyncJob en Serilog. Usar bloqueo manual si es urgente
El job deja de sincronizar después de un tiempo Token expirado y auto-retry fallando (GeoVictoria no disponible temporalmente) Limen reintentará automáticamente en el próximo ciclo. Si persiste, verificar que GeoVictoria está en línea y que las credenciales siguen válidas
Credenciales no se pueden descifrar La clave DataProtection cambió (nuevo servidor o deployment limpio) Ir a Integraciones y volver a guardar el ConsumerSecret con la clave actual
