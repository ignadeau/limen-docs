BUK INTEGRATION

Sincronización en Tiempo Real mediante Webhooks

1. Arquitectura de Integración

La integración con Buk es de alta prioridad y utiliza Webhooks en tiempo real. Esto permite que el sistema reaccione instantáneamente ante bajas laborales.

2. Proceso de Autenticación

Limen utiliza autenticación simple y segura para las peticiones salientes:

  • Header Obligatorio: auth_token: {ApiKey} en cada request.
  • Endpoint de Nómina: GET /employees/active (descarga inicial de empleados).

3. Webhooks de Baja (Real-Time Bloqueo)

Buk envía una notificación a Limen cuando un contrato finaliza. Para configurar esto, debe ingresar la siguiente URL en el panel de Webhooks de Buk:

https://api.limen.app/api/webhooks/buk
Nota: Se debe incluir el header X-Tenant-Id con el ID de su empresa en Limen.

Seguridad HMAC-SHA256

Limen protege el endpoint de webhooks verificando la autenticidad de Buk:

  • Header de Firma: X-Buk-Signature.
  • Validación: Limen realiza una comparación de tiempo constante (Constant-time comparison) para evitar ataques de temporización.
  • Acción: Ante un evento válido de baja, se dispara el proceso ProcessTerminationAsync, bloqueando el acceso en el hardware en menos de 30 segundos.
