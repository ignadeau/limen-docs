1. Qué es BioStar 2 y cómo lo usa Limen

Suprema BioStar 2 es una plataforma de gestión de control de acceso y tiempo/asistencia de la empresa coreana Suprema. Se instala on-premise en un servidor del cliente y gestiona dispositivos de la familia Suprema (BioEntry, BioStation, FaceStation, etc.).

Limen se integra con BioStar 2 a través de su REST API local expuesta por el servidor BioStar 2. Limen actúa como cliente de la API — consulta puertas, sincroniza personas, bloquea/desbloquea credenciales y descarga eventos de acceso. No reemplaza a BioStar 2; lo complementa con portales de autogestión, flujos de aprobación y dashboards.

Nota: Limen usa la API REST interna de BioStar 2 (la misma que usa el cliente web de BioStar 2). No requiere ningún SDK ni plugin adicional de Suprema.

2. Prerequisitos

  • BioStar 2 versión 2.8 o superior (la API REST está disponible desde v2.x; se recomienda v2.8+ para mayor estabilidad).
  • Servidor BioStar 2 accesible desde la red donde corre Limen (mismo segmento LAN o VPN). Puerto por defecto: 443 (HTTPS).
  • Licencia de BioStar 2 activa. La API REST no requiere licencia adicional — viene incluida con la licencia base.
  • Usuario administrador de BioStar 2 con permisos suficientes (ver sección 3).
  • El servicio "API Server" de BioStar 2 debe estar habilitado (ver sección 4).
Certificado SSL: La mayoría de las instalaciones de BioStar 2 on-premise usan un certificado auto-firmado. Limen acepta certificados auto-firmados en la comunicación con BioStar 2. Para producción, se recomienda instalar un certificado de una CA reconocida en el servidor BioStar 2.

3. Crear usuario de integración en BioStar 2

Se recomienda crear un usuario dedicado para la integración con Limen en lugar de usar el usuario admin principal, para facilitar la auditoría y el control de acceso.

  1. Inicia sesión en BioStar 2 con el usuario administrador principal (normalmente en https://<ip-servidor>).
  2. Ve a Settings → Accounts → Administrator.
  3. Haz clic en Add para crear un nuevo administrador.
  4. Completa los campos:
    • Login ID: p. ej. limen-integration
    • Password: usa una contraseña robusta (mínimo 12 caracteres, mezcla de letras, números y símbolos).
    • Operator Level: selecciona Administrator o crea un nivel personalizado con los permisos que se detallan abajo.
  5. Guarda el usuario.

4. Habilitar la API REST en BioStar 2

BioStar 2 tiene un servicio de API que debe estar activo para que Limen pueda comunicarse con él.

  1. En BioStar 2, ve a Settings → Server.
  2. Busca la sección "API Server" o "HTTP API".
  3. Activa la opción Enable si no está activa.
  4. Verifica el puerto configurado (por defecto 443 para HTTPS o 80 para HTTP). Se recomienda HTTPS.
  5. Guarda los cambios y reinicia el servicio de BioStar 2 si el sistema lo solicita.
Verificación rápida: abre un navegador y navega a https://<ip-servidor>/api/login. Deberías ver un error HTTP 405 (Method Not Allowed) o una respuesta JSON vacía — eso confirma que el API Server está activo. Si ves "página no encontrada", el servicio no está habilitado o el puerto es distinto.

5. Configurar en Limen

En Limen, ve a Administración → Integraciones y en la sección Control de Acceso Físico selecciona el proveedor Suprema BioStar 2.

CampoDescripciónEjemplo
URL Base Dirección completa del servidor BioStar 2, incluyendo esquema (https://) y sin barra al final. https://192.168.1.100
Usuario Login ID del usuario de integración creado en el paso 3. limen-integration
Contraseña Contraseña del usuario de integración. —
Seguridad: Las credenciales se cifran con AES-256 (DataProtection de .NET) antes de almacenarse en la base de datos de Limen. Nunca se guardan en texto plano ni aparecen en los logs. Una vez guardadas, el campo contraseña muestra ••••••••; para cambiarla debes ingresar el nuevo valor y guardar nuevamente.

7. Mapeo de puertas

En Limen, las puertas se identifican por su Hardware ID, que en BioStar 2 corresponde al campo id de cada puerta (un número entero asignado por BioStar 2).

Cómo obtener el ID de una puerta en BioStar 2

  1. En Limen, ve a Administración → Puertas y usa el botón "Importar desde hardware". Limen listará todas las puertas con su ID.
  2. Alternativamente, en BioStar 2 ve a Device → Door y selecciona una puerta. La URL del navegador mostrará el ID: /door/edit/42.
  3. También puedes consultarlo directamente via API: 
    GET https://<ip-servidor>/api/doors?limit=100&offset=0
Header: bs-session-id: {tu-session-id}
    La respuesta incluye el id de cada puerta.

8. Cómo funciona la autenticación

BioStar 2 usa autenticación basada en sesión con un token llamado bs-session-id:

  1. Limen llama a POST /api/login con usuario y contraseña.
  2. BioStar 2 responde con un session_id en el body JSON (User.session_id) y también en el header Set-Cookie: bs-session-id=....
  3. Limen cachea el session_id en memoria con TTL de 50 minutos (las sesiones de BioStar 2 expiran a los 60 min por defecto).
  4. Todas las requests siguientes incluyen el header bs-session-id: {session_id}.
  5. Si el servidor responde con HTTP 401 (sesión expirada), Limen limpia el cache y re-autentica automáticamente una sola vez.
Sin estado en el servidor: El cache del session_id vive en la memoria del proceso Limen. Si Limen se reinicia, el próximo request al hardware BioStar 2 realizará un nuevo login automáticamente.

9. Troubleshooting

SíntomaCausa probableSolución
Logs: "Login falló: HTTP 401" Credenciales incorrectas Verifica usuario y contraseña en BioStar 2 → Settings → Accounts.
Logs: "Login falló: HTTP 404" La API no está habilitada o la URL base es incorrecta Verifica la URL y que API Server esté activo en BioStar 2 → Settings → Server.
Timeout en login Servidor no accesible (red, firewall) Comprueba conectividad: ping <ip-servidor> y que el puerto 443 esté abierto.
Error SSL / certificado Certificado auto-firmado Limen acepta certificados auto-firmados por defecto. Si persiste, revisa que la URL comience con https:// y que el servidor esté accesible.
Logs: "Login exitoso pero no se obtuvo session_id" Versión muy antigua de BioStar 2 con formato de respuesta distinto Actualiza BioStar 2 a v2.8+. Contacta a soporte Limen si el problema persiste.
Logs: "usuario PIN X no encontrado en BioStar 2" en BlockUser El usuario no fue sincronizado previamente a BioStar 2 Ejecuta primero SyncPerson para el usuario. En el flujo normal de Limen, la sincronización ocurre al onboardear a la persona.
Apertura remota de puerta falla con 404 Versión de BioStar 2 que usa endpoint alternativo Limen prueba automáticamente el endpoint de fallback. Si ambos fallan, verifica que el ID de puerta sea correcto y que el usuario tenga permiso "Door Control".
No llegan eventos de acceso El rango de fechas no retorna datos o formato incorrecto Verifica en BioStar 2 → Monitoring → Event Log que haya eventos en el período consultado. Revisa que el timezone del servidor BioStar 2 esté configurado correctamente.

10. Limitaciones conocidas

FuncionalidadEstadoAlternativa
Registro de patentes LPR No disponible La API REST local de BioStar 2 no incluye endpoints de LPR. Para LPR con hardware Suprema, se requiere integración con el módulo Suprema LPR o con un sistema LPR externo. Limen registra un warning en logs y continúa sin error.
Enrolamiento de huella dactilar / iris No disponible La API REST de BioStar 2 no permite enrolar datos biométricos remotamente por razones de seguridad. El enrolamiento debe hacerse desde el dispositivo físico o desde la UI de BioStar 2.
Push de eventos en tiempo real Polling Limen usa polling cada 30 segundos para descargar eventos nuevos. BioStar 2 soporta webhooks/callbacks en versiones más recientes; esta funcionalidad está planificada para una versión futura de Limen.
Gestión de tarjetas de acceso Parcial Limen gestiona las credenciales lógicas (habilitado/deshabilitado, grupos de acceso) pero no emite ni revoca tarjetas físicas. La gestión de tarjetas se hace desde la UI de BioStar 2.
Multi-servidor BioStar 2 Uno por tenant Cada tenant de Limen se conecta a un único servidor BioStar 2. Si el cliente tiene varios servidores BioStar 2 independientes, se puede configurar uno por tenant en Limen. Contacta a soporte para este escenario.
¿Todo configurado? Regresa a Administración → Integraciones en Limen, ingresa las credenciales y haz clic en "Probar conexión" para verificar que la integración funciona correctamente.
