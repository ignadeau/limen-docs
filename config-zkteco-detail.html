Mapeo de Puertas (Hardware → Limen)

Cada puerta física en ZKBio tiene un identificador único (DoorID). Limen usa ese ID para enviar comandos de apertura, recibir eventos y mostrar reportes. El mapeo se hace en Administración → Infraestructura → Puertas.

Opción A — Importar automáticamente desde el hardware

  1. Abre el diálogo Nueva Puerta.
  2. En la sección Importar desde hardware, haz clic en Listar puertas del hardware.
  3. Limen consultará el endpoint /api/door/list de ZKBio y mostrará todas las puertas disponibles.
  4. Haz clic en Usar este ID junto a la puerta deseada. El campo "ID en hardware" se completa automáticamente con el valor correcto.
  5. Asigna la zona y el nombre dentro de Limen, luego guarda.

Opción B — Ingresar el ID manualmente

  1. En ZKBio, ve a Access → Device Management → Door Setting.
  2. Anota el valor de la columna Serial Number de la puerta (ej: VDE2242400041). Ese es el ID que ZKBio usa internamente para identificar la puerta en la API.
  3. En Limen, crea la puerta e ingresa ese valor en el campo ID en hardware.
⚠️ El ID en hardware es sensible. Si usas un ID incorrecto, los comandos de apertura remota llegarán a la puerta equivocada (o no llegarán). Siempre verifica con "Listar puertas" o consultando directamente en el panel ZKBio.

Sincronización Automática de Eventos

Limen corre un job en background (AccessEventSyncJob) cada 30 segundos que consulta el endpoint /api/transaction/monitor de ZKBio para traer los eventos de acceso nuevos (entradas, salidas, accesos denegados).

ComportamientoDetalle
Ventana inicialSi no hay eventos previos en BD, retrocede 2 horas para no perder eventos recientes al activar.
OverlapEn cada ciclo se retrocede 2 minutos sobre el último evento para capturar eventos que el hardware entrega con retraso.
DeduplicaciónEl índice único IX_AccessEvents_TenantDedup en BD evita duplicados incluso en caso de reinicio o concurrencia.
Errores de cifradoSi las credenciales no se pueden descifrar (clave DataProtection rotada), se registra un warning una sola vez y se usa el simulador hasta que se guarden nuevamente.

Troubleshooting — Diagnóstico de Problemas

Síntoma: "No se encontraron puertas en el hardware"

Revisa los logs del servidor (nivel Warning). Los mensajes clave son:

Mensaje en LogCausaSolución
Sin tenant resuelto → simulador No hay tenant asociado a la sesión actual Verificar que el usuario tiene un tenant asignado y que el subdominio está bien configurado
sin config de AC (o Provider=None) La integración ZKBio no fue guardada en este tenant Ir a Integraciones → Control de Acceso → seleccionar ZKBio → guardar
ZkBio configurado pero sin API Key Se guardó la URL pero no la API Key Editar la integración e ingresar la API Key generada en ZKBio API Management
HTTP 404 en todos los endpoints La URL base apunta al servidor correcto pero la API no está habilitada, o la API Key es inválida y el servidor retorna 404 en lugar de 401 Ver sección "Verificar API Key" más abajo
devolvió HTML La URL base apunta al portal web de ZKBio en lugar de la API, o la API Key es completamente incorrecta Verificar que la URL no incluye rutas extra (no debe terminar en /login, /index, etc.)
code=-1 o mensaje de "No permission" La API Key existe pero no tiene permisos para el módulo de puertas En ZKBio API Management, editar el token y agregar los permisos de Access Control

Cómo verificar la API Key manualmente

Puedes probar la key directamente desde el navegador o con curl:

# Desde el navegador (reemplazar con tu URL y key):
https://zkbio.tuempresa.com/api/door/list?pageNo=1&pageSize=10&access_token=TU_API_KEY

# Con curl:
curl "https://zkbio.tuempresa.com/api/door/list?pageNo=1&pageSize=10&access_token=TU_API_KEY"

La respuesta esperada es un JSON con "code": 0. Si recibes HTML o "code": -1, la key está mal o no tiene permisos.

Otros errores frecuentes

ErrorCausa probableSolución
Timeout / conexión rechazada Firewall bloqueando el puerto, o el servidor ZKBio está apagado Verificar que el servidor está en línea y que el puerto (por defecto 8098 o 443) está abierto desde el servidor de Limen
SSL error / certificado inválido ZKBio usa HTTPS con certificado autofirmado Configurar la URL con http:// si el servidor no tiene certificado válido, o instalar el certificado raíz en el servidor de Limen
Eventos que no aparecen El job sincroniza cada 30s; puede haber hasta 30s de latencia Esperar un ciclo completo. Si persiste, revisar logs de AccessEventSyncJob
Credenciales no se pueden descifrar La clave DataProtection cambió (ej: nuevo servidor, deployment limpio) Ir a Integraciones y volver a guardar las credenciales con la clave actual
