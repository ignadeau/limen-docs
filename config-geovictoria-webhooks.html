Por qué GeoVictoria no tiene Webhooks en Limen

⚠️ Estado actual: El conector de GeoVictoria en Limen opera exclusivamente con Polling Periódico. No existe un endpoint de webhook activo para GeoVictoria. La página anterior de este tutorial documentaba funcionalidad no implementada — esta versión refleja el estado real del sistema.

Existen dos razones para esta situación:

RazónDetalle
1. Soporte de GeoVictoria La API pública de GeoVictoria (versión utilizada en este conector) no incluye un mecanismo nativo de webhooks salientes para eventos de baja laboral. GeoVictoria puede tener esta funcionalidad en planes enterprise o versiones más recientes — verificar con el ejecutivo de cuenta de GeoVictoria.
2. Prioridad de desarrollo El conector actual cubre el caso de uso principal (sincronización de nómina). El webhook de tiempo real está en el roadmap pero aún no implementado en Limen. Si GeoVictoria de tu cliente lo soporta y es prioritario, es un desarrollo de algunas horas — ver sección "Implementar Webhooks" más abajo.

Cómo Operar sin Webhooks

Con GeoVictoria hay una ventana de hasta 15 minutos entre una baja en RRHH y el bloqueo en hardware. Estas son las estrategias para mitigar ese riesgo:

Opción A — Bloqueo manual inmediato (recomendado para bajas críticas)

  1. En el momento del finiquito, el área de RRHH o Seguridad informa a la guardia
  2. El operador de Limen va a Personas → [nombre del empleado] → Bloquear acceso
  3. El bloqueo actúa sobre el hardware (ZK/Hikvision) en 2–5 segundos
  4. No requiere esperar el ciclo de polling

Opción B — Reducir el intervalo de polling

El job de sincronización corre cada 15 minutos por defecto. Si el cliente requiere una ventana menor, el intervalo es configurable por tenant. Reducirlo a 5 minutos es razonable sin impacto de performance para nóminas de hasta 5.000 empleados. Contactar al equipo de Limen para ajustarlo.

Opción C — Migrar a Buk

Si el cliente usa GeoVictoria solo para asistencia y tiene contratado Buk para liquidaciones, puede configurar Buk como proveedor de RRHH en Limen (en lugar de GeoVictoria) y obtener webhooks en tiempo real para bajas laborales. Ambos sistemas son mutuamente excluyentes como proveedor de RRHH en Limen.

Implementar Webhooks de GeoVictoria (Roadmap)

Si GeoVictoria del cliente soporta webhooks salientes, el desarrollo en Limen requiere los siguientes pasos. Tiempo estimado: 4–8 horas.

  1. Confirmar con GeoVictoria el formato del payload, el mecanismo de autenticación (¿Bearer Token estático? ¿HMAC-SHA256? ¿IP Whitelist?) y el nombre exacto del evento de baja (EMPLOYEE_TERMINATED, work_ended, etc.)
  2. Crear GeoVictoriaWebhookProcessor en Limen.Infrastructure/HrSystem/ siguiendo el mismo patrón que BukWebhookProcessor: verificación de autenticación, parseo de payload, despacho a IIdentitySyncService.ProcessTerminationAsync()
  3. Registrar el endpoint en Limen.Web/Program.cs:
    app.MapPost("/api/webhooks/geovictoria", async (...) => { ... });
  4. Agregar GeoVictoriaWebhookSecret a GeoVictoriaConfig en IntegrationConfigs.cs y al formulario de Integraciones en la UI
  5. Actualizar este tutorial con el paso a paso de configuración en el panel de GeoVictoria
Referencia de implementación: El webhook de Buk está completamente implementado y es el modelo a seguir. Ver src/Limen.Infrastructure/HrSystem/BukWebhookProcessor.cs y su registro en src/Limen.Web/Program.cs.
