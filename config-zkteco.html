Modelo de Autenticación

ZKBio CVSecurity autentica mediante una API Key estática que se genera una sola vez en el panel de administración del servidor ZK. Esta key no expira a menos que se regenere manualmente, y se envía como query parameter en cada llamada:

GET /api/door/list?pageNo=1&pageSize=500&access_token=TU_API_KEY
⚠️ Importante: A diferencia de otros sistemas, ZKBio NO usa usuario + contraseña para la API REST. Usar credenciales de usuario en este campo producirá errores 404 o respuestas HTML en lugar de JSON.
